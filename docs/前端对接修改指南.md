# 前端对接修改指南

> 本文档列出前端 `index.html` 需要配合后端 API 做出的修改。
> 后端 API 路径和字段已按 `docs/API文档.md` 和 `docs/api-spec.yaml` 统一。

---

## 1. 登录接口字段名

后端 `POST /api/auth/login` 接收的字段是 `email` 和 `password`。

**当前代码（约第 1477 行）：**
```javascript
body: JSON.stringify({ username, password })
```

**改为：**
```javascript
body: JSON.stringify({ email: username, password })
```

---

## 2. 注册接口字段名

后端 `POST /api/auth/register` 接收的字段是 `email`、`password`、`nickname`（必填）。

**当前代码（约第 1522 行）：**
```javascript
body: JSON.stringify({ username, password })
```

**改为：**
```javascript
body: JSON.stringify({ email: username, password, nickname: username })
```

> 建议后续在注册表单中增加昵称输入框，让用户自定义昵称。

---

## 3. 响应数据解包

后端所有接口统一返回格式：
```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

列表接口的 `data` 结构为：
```json
{
  "list": [...],
  "total": 10,
  "page": 1,
  "pageSize": 10
}
```

### 3.1 loadProducts()（约第 745 行）

**当前：**
```javascript
products = await response.json();
```

**改为：**
```javascript
const result = await response.json();
const rawList = result.data && result.data.list ? result.data.list : [];
products = rawList.map(p => ({
    ...p,
    mode: p.tradeMode || 'cash',
    cash: p.cashPrice || 0,
    points: p.pointsPrice || 0,
    cond: p.condition || '',
    desc: p.description || '',
    date: p.createdAt || '',
    blob: ['pink','mint','lavender'][p.id % 3]
}));
```

### 3.2 loadWants()（约第 779 行）

**当前：**
```javascript
wants = await response.json();
```

**改为：**
```javascript
const result = await response.json();
wants = (result.data && result.data.list ? result.data.list : []).map(w => ({
    ...w,
    desc: w.description || '',
    date: w.createdAt || ''
}));
```

> 注意：求购发布者对象字段为 `w.user.nickname`（不是 `w.user.name`）。

### 3.3 loadPointsLog()（约第 793 行）

**当前：**
```javascript
pointsLog = await response.json();
```

**改为：**
```javascript
const result = await response.json();
pointsLog = (result.data && result.data.list ? result.data.list : []).map(l => ({
    t: l.description,
    v: l.amount,
    d: l.createdAt
}));
```

### 3.4 loadMyTrades()（约第 806 行）

**当前：**
```javascript
myTrades = await response.json();
```

后端返回的是统一分页列表（每条记录有 `type: "buy"` 或 `"sell"`），不是 `{buy:[...], sell:[...]}` 分组。需要前端自行分组：

**改为：**
```javascript
const result = await response.json();
const allTrades = result.data && result.data.list ? result.data.list : [];
myTrades = {
    buy: allTrades.filter(t => t.type === 'buy'),
    sell: allTrades.filter(t => t.type === 'sell')
};
```

每条交易记录的字段：`id`、`productId`、`product`（对象，含 `id`/`title`/`icon`/`category`/`tradeMode`）、`type`、`status`、`tradeDate`。

## 4. 字段名映射

后端返回的字段与前端 `renderGrid()` 使用的字段名不同：

| 后端字段 | 前端期望字段 | 说明 |
|---------|------------|------|
| `tradeMode` | `mode` | 值: cash / points / both |
| `cashPrice` | `cash` | 现金价格（数字） |
| `pointsPrice` | `points` | 积分价格（数字） |
| `condition` | `cond` | 成色描述 |
| `description` | `desc` | 商品描述 |
| `createdAt` | `date` | 创建时间字符串 |
| `seller.nickname` | `seller.name` | 卖家昵称 |
| （无） | `blob` | 卡片颜色，可用 `['pink','mint','lavender'][id % 3]` 生成 |

建议在 `loadProducts()` 中做一层 map 转换（见 3.1）。

## 5. submitPublish() 函数参数缺失

HTML 按钮调用的是 `submitPublish()`（无参数），但函数签名是 `submitPublish(formData, type)`。

**修复方式：** 在函数开头加入表单数据收集逻辑：

```javascript
async function submitPublish(formData, type) {
    if (!formData) {
        const pubMode = document.getElementById('segWant').classList.contains('active') ? 'want' : 'idle';
        type = pubMode === 'idle' ? 'product' : 'want';

        const title = document.getElementById('pTitle').value.trim();
        const desc = document.getElementById('pDesc').value.trim();
        if (!title) { alert('请填写标题'); return; }

        if (type === 'product') {
            const cond = document.querySelector('#condRow .opt-chip.sel');
            const mode = document.querySelector('#modeRow .mode-box.sel');
            formData = {
                title: title,
                description: desc,
                condition: cond ? cond.dataset.c : '99新',
                tradeMode: mode ? mode.dataset.m : 'cash',
                cashPrice: parseFloat(document.querySelector('#cashBox input')?.value) || 0,
                pointsPrice: parseInt(document.querySelector('#pointsBox input')?.value) || 0,
                category: '其他',
                icon: 'box'
            };
        } else {
            formData = {
                title: title,
                description: desc,
                budget: '面议',
                icon: 'star'
            };
        }
    }
    // ... 后续发请求的代码不变
}
```

## 6. 收藏操作

后端收藏接口已拆分为两个独立操作：

- `POST /api/favorites/{productId}` — 添加收藏
- `DELETE /api/favorites/{productId}` — 取消收藏
- `GET /api/favorites/check/{productId}` — 检查是否已收藏

前端当前使用 `POST` toggle，需改为根据当前状态选择 POST 或 DELETE。

## 7. 后端 API 路径汇总

所有路径基于 `context-path: /api`，前端请求时加 `/api` 前缀。

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /api/auth/register | 注册 | 否 |
| POST | /api/auth/login | 登录 | 否 |
| GET | /api/user/profile | 获取个人信息 | 是 |
| PUT | /api/user/profile | 更新个人信息 | 是 |
| GET | /api/products | 商品列表（支持 category/tradeMode/status/keyword/page/pageSize） | 否 |
| GET | /api/products/{id} | 商品详情 | 否 |
| POST | /api/products | 发布商品 | 是 |
| PUT | /api/products/{id}/status | 更新商品状态（body: `{"status":"xxx"}`) | 是 |
| GET | /api/user/products | 我的商品 | 是 |
| GET | /api/wants | 求购列表 | 否 |
| GET | /api/wants/{id} | 求购详情 | 否 |
| POST | /api/wants | 发布求购 | 是 |
| PATCH | /api/wants/{id}/close | 关闭求购 | 是 |
| GET | /api/user/wants | 我的求购 | 是 |
| GET | /api/user/trades | 我的交易（支持 type=buy/sell） | 是 |
| GET | /api/user/points | 积分余额 | 是 |
| GET | /api/user/points/log | 积分流水 | 是 |
| GET | /api/favorites | 收藏列表 | 是 |
| POST | /api/favorites/{productId} | 添加收藏 | 是 |
| DELETE | /api/favorites/{productId} | 取消收藏 | 是 |
| GET | /api/favorites/check/{productId} | 检查是否已收藏 | 是 |

## 8. Token 使用说明

- 登录成功后，后端返回 `result.data.token`，需存入 `localStorage`
- 所有需要认证的接口，请求头加：`Authorization: Bearer <token>`
- `authFetch()` 封装已正确处理这部分，无需额外修改

## 9. 静态文件访问

`index.html` 放在 `herloop-backend/src/main/resources/static/` 下，
启动后端后通过 `http://localhost:8080/api/index.html` 访问。
